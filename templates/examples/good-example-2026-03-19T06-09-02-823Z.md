## システムの仕組み解説
![システムの仕組み解説](images/chapter-01.jpg)

ここでは、YouTubeリサーチ自動化システムの裏側、つまりどのようにしてこの仕組みが動いているのかを具体的に解説します。

**システム全体の流れ**

1.  **スプレッドシートでの条件設定**：
    -   共有するスプレッドシートの「設定」タブで、リサーチしたい動画の条件を細かく指定します。
    -   **設定可能な条件の例**：
        -   **エリア**：日本、日本以外、全世界など。
        -   **動画の長さ**：ショート動画、ミディアム動画、ロング動画（20分以上）など。
        -   **再生数と登録者数の関係**：登録者数よりも再生数が上回っている動画（伸びる動画の要素を見つけやすい）。
        -   **再生数と登録者数の乖離**：再生数が登録者数の2倍以上あるか。
        -   **再生数の閾値**：1,000回以上、10,000回以上など。
    -   これらの条件は、**プロンプト**を使って一括でスプレッドシートに貼り付けることも可能です。
2.  **n8nによる自動化**：
    -   設定された条件をもとに、**n8n**がYouTubeから情報を自動で取得します。
    -   n8nは、複数のブロック（ノード）を組み合わせてワークフローを作成するノーコード自動化ツールです。
    -   **実行方法**：
        -   **手動実行**：必要に応じて手動でリサーチを実行できます。
        -   **定期実行**：1日1回、1週間に1回、1ヶ月に1回など、設定した周期で自動的にリサーチを実行し、スプレッドシートに結果を反映させます。
    -   今回配布する**n8nテンプレート**をインポートし、一部設定するだけで同じ仕組みを再現できます。
    -   n8nは**無料プラン**から利用できるため、今回の仕組みは完全無料で構築可能です。

💡 **このセクションのポイント**

- **スプレッドシート**：リサーチ条件（エリア、動画長、再生数/登録者数比など）を詳細に設定
- **プロンプト**：複雑な条件設定も、プロンプトを活用することで一括で簡単に設定できる
- **n8n**：スプレッドシートの条件に基づき、YouTube情報を自動取得・整理するノーコードツール

## 構築開始
![構築開始](images/chapter-03.jpg)

それでは、いよいよYouTubeリサーチ自動化システムの構築を始めましょう。

### Step 1: n8nアカウントの作成とログイン

まずはn8nの公式サイトにアクセスします。

1.  **Googleで「n8n」を検索**し、公式サイトをクリックします。
2.  **ログイン/アカウント作成**：
    -   初めての方は「Get Started」から、Googleアカウントなどでアカウントを作成してください。
    -   既存アカウントをお持ちの方は「Sign In」からログインします。
3.  **プラン選択（推奨）**：
    -   n8nはクラウド版（月額20ユーロ〜）とセルフホスト版（無料）があります。
    -   手軽に始めたい方は、まずはクラウド版の**トライアル期間**を利用してメールアドレスで登録するのがおすすめです。

### Step 2: n8nワークフローのインポート

次に、配布されたn8nテンプレートをインポートします。

1.  n8nにログイン後、右上の「**Create Workflow**」ボタンをクリックします。
2.  このnote記事の冒頭またはYouTube概要欄に記載されている**noteのURL**にアクセスします。
3.  noteページ内の「**n8n用のJSONファイルはこちら**」というリンクから、テンプレートファイルをダウンロードします。
4.  n8nのワークフロー画面に戻り、右上の「**…（点が3つ並んだアイコン）**」をクリックし、「**Import from File**」を選択します。
5.  ダウンロードしたJSONファイルを指定し、インポートします。
6.  インポートが成功すると、ワークフローが画面に表示されます。

### Step 3: スプレッドシートのコピーと連携設定

次に、リサーチ結果を保存するスプレッドシートを準備し、n8nと連携させます。

1.  再度、note記事またはYouTube概要欄の**noteのURL**にアクセスします。
2.  noteページ内の「**コピー用スプレッドシートはこちら**」というリンクをクリックします。
3.  表示されたスプレッドシートは直接編集できないため、左上の「**ファイル**」メニューから「**コピーを作成**」を選択し、ご自身のGoogleアカウントで編集可能なコピーを作成してください。
4.  n8nのワークフロー画面に戻り、以下の2つのノードを設定します。

    #### 3-1. 「設定シート読み込み」ノードの設定
    1.  「**設定シート読み込み**」ノードをダブルクリックして開きます。
    2.  **Credential（認証情報）の設定**：
        -   初めてGoogleアカウントを連携する場合は、「**Create New Credential**」をクリックします。
        -   「**Sign in with Google**」を選択し、ご自身のGoogleアカウントでログインします。
        -   途中で表示されるアクセス許可の確認画面では、すべて「許可」または「続行」を選択し、チェックボックスがあればすべてチェックを入れてください。
        -   連携が完了すると、Credentialの選択肢にご自身のGoogleアカウントが表示されますので、そちらを選択します。
    3.  **Document & Sheetの設定**：
        -   「Document」欄で「From List」を選択し、先ほどコピーしたスプレッドシート（例: 「撮影用」など、ご自身でつけた名前）を選択します。
        -   「Sheet」欄で「From List」を選択し、「**設定**」タブを選択します。
    4.  設定が完了したら、ノードを閉じます。

    #### 3-2. 「スプレッドシートに保存」ノードの設定
    1.  ワークフローの最後にある「**スプレッドシートに保存**」ノードをダブルクリックして開きます。
    2.  **Credentialの設定**：
        -   先ほど作成したGoogleアカウントのCredentialをそのまま選択します。
    3.  **Document & Sheetの設定**：
        -   「Document」欄で、先ほど選択したスプレッドシートと同じもの（例: 「撮影用」）を選択します。
        -   「Sheet」欄で「From List」を選択し、「**リサーチ結果**」タブを選択します。
    4.  **カラムリストのリフレッシュ**：
        -   まれに、スプレッドシートを連携した際に、デフォルトで設定されていた項目が空になることがあります。その場合は、ノード内の右上の「**…（点が3つ並んだアイコン）**」をクリックし、「**Refresh Column List**」を選択してください。これにより、スプレッドシートの列名や項目が正しく読み込まれます。
    5.  **関数が消えた場合の対応**：
        -   もし、個々の項目に設定されていた関数が消えてしまった場合は、note記事内の「**valuto度が消えてしまった時のメモ**」セクションを参照し、記載されている関数をコピーして貼り付けてください。
    6.  設定が完了したら、ノードを閉じます。

### Step 4: スプレッドシートのリサーチ条件設定

コピーしたスプレッドシートで、リサーチ条件を設定します。

1.  コピーしたスプレッドシートを開き、「**設定**」タブに移動します。
2.  note記事内の「**リサーチ設定作成プロンプト**」をコピーします。
3.  **ChatGPT、Claude、Geminiなど**、お好みのAIチャットツールにプロンプトを貼り付けます。
4.  AIから「リサーチしたいジャンルやテーマを教えてください」と聞かれたら、ご自身がリサーチしたいキーワード（例: 「キャリア系」）を入力します。
5.  AIが生成したリサーチ条件をコピーし、スプレッドシートの「設定」タブに貼り付けます。
6.  必要に応じて、エリアや動画の長さなど、詳細な条件をプルダウンメニューから手動で調整することも可能です。

### Step 5: YouTube Data APIキーの取得と設定

n8nがYouTubeの情報を取得できるように、APIキーを設定します。

1.  **Googleで「Google Cloud コンソール」を検索**し、公式サイトをクリックしてコンソールを開きます。
2.  右上の「**コンソール**」ボタンをクリックします。
3.  **プロジェクトの選択/作成**：
    -   左上のプロジェクト名が表示されている部分をクリックし、「新しいプロジェクト」を選択して新規プロジェクトを作成するか、既存のプロジェクトを選択します。
    -   プロジェクト作成後、そのプロジェクトが選択されていることを確認してください。
4.  左側のメニューバーから「**APIとサービス**」→「**ライブラリ**」をクリックします。
5.  検索窓で「**YouTube**」と入力し、「**YouTube Data API v3**」を選択してクリックします。
6.  「**有効にする**」ボタンが表示されている場合は、クリックしてAPIを有効にします。（すでに有効な場合は「有効です」と表示されます）
7.  再度、左側のメニューバーから「**APIとサービス**」→「**認証情報**」をクリックします。
8.  **APIキーの作成**：
    -   初めてAPIキーを作成する場合は、「**認証情報を作成**」をクリックし、「**APIキー**」を選択します。
    -   名前は任意で構いません。アプリケーションの制限、APIの制限は「なし」のままで大丈夫です。「作成」をクリックします。
    -   すでにAPIキーがある場合は、表示されているAPIキーをクリックし、「鍵を表示します」を選択してコピーします。
9.  作成されたAPIキーをコピーします。
10. n8nのワークフロー画面に戻り、以下の3つのノードにAPIキーを貼り付けます。

    #### 5-1. 「YouTube検索」ノード
    1.  「**YouTube検索**」ノードをダブルクリックして開きます。
    2.  下にスクロールし、「Name」が「key」となっている行の「**Value**」欄に、コピーしたYouTube APIキーを貼り付けます。
    3.  ノードを閉じます。

    #### 5-2. 「動画統計取得」ノード
    1.  「**動画統計取得**」ノードをダブルクリックして開きます。
    2.  同様に「Name」が「key」となっている行の「**Value**」欄に、コピーしたYouTube APIキーを貼り付けます。
    3.  ノードを閉じます。

    #### 5-3. 「チャンネル情報取得」ノード
    1.  「**チャンネル情報取得**」ノードをダブルクリックして開きます。
    2.  同様に「Name」が「key」となっている行の「**Value**」欄に、コピーしたYouTube APIキーを貼り付けます。
    3.  ノードを閉じます。

### Step 6: 定期実行の設定

最後に、リサーチを自動で実行する周期を設定します。

1.  ワークフローの最初にある「**定期実行**」ノードをダブルクリックして開きます。
2.  「Mode」欄で、リサーチを実行したい周期（例: 「Every Day（毎日）」「Every Week（毎週）」「Every Month（毎月）」など）を選択します。
3.  設定が完了したら、ノードを閉じます。

💡 **このセクションのポイント**

- **n8nログイン**：クラウド版のトライアルから手軽に開始できる
- **ワークフローインポート**：noteからJSONテンプレートをダウンロードし、n8nにインポートする
- **スプレッドシート連携**：Googleアカウント認証後、「設定」タブと「リサーチ結果」タブをそれぞれn8nノードに紐付け
- **リサーチ条件設定**：AIプロンプトを活用してスプレッドシートに一括設定し、手動で微調整も可能
- **YouTube Data APIキー**：Google Cloud ConsoleでAPIを有効化し、APIキーを取得してn8nの3つのYouTubeノードに設定
- **定期実行**：n8nの「定期実行」ノードで、リサーチの頻度を「毎日」「毎週」などで設定

## 補足:重複チェック
![補足:重複チェック](images/chapter-05.jpg)

定期実行でリサーチを続けると、スプレッドシートの「リサーチ結果」タブには同じ動画が複数回記録される可能性があります。ここでは、重複したデータを効率的に管理する方法と、AIを活用した分析のヒントを解説します。

**1. 重複データの確認と管理**

-   **「重複チェック」タブの活用**：スプレッドシートには「**重複チェック**」という専用タブが用意されています。このタブを開くと、取得された動画データの中から重複するものを自動で除外し、ユニークな動画のみが表示されるようになっています。
-   **日常的な確認**：リサーチ結果を確認する際は、この「重複チェック」タブを見ることで、常に最新かつ重複のないデータで市場を把握できます。

**2. AIを活用した分析**

収集された膨大なデータは、AIの力を借りることでさらに価値のある情報に変換できます。

-   **Gemini連携（課金者向け）**：Geminiに課金している方は、スプレッドシートの右上に表示されるGeminiマークから、チャット形式でデータ分析を依頼できます。「分析レポートを出して」といった指示を出すだけで、自動でレポートが生成されます。
-   **外部AIツールとの連携**：Geminiに課金していない場合でも、スプレッドシートのデータをExcelなどにエクスポートし、ChatGPTやClaudeなどのAIチャットツールに読み込ませることで、同様の分析が可能です。
    -   「このデータから、伸びている動画の共通点を教えてください」
    -   「特定のキーワードで、再生数が多い動画の傾向を分析してください」
    -   「次の動画企画に繋がりそうなインサイトを抽出してください」
