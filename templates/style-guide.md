# note記事プロ仕様スタイルガイド

## note で伸びる記事の鉄則

### 1. 冒頭3行で読者を引き込む
- 「あなたはこんな悩みを抱えていませんか？」で共感を獲得
- 「この記事を読むと〇〇ができるようになります」で期待値を設定
- 具体的な数字・固有名詞を冒頭に入れる（「5分で」「ElevenLabsを使って」など）

### 2. 目次を必ず入れる
記事の冒頭（はじめにの直後）に目次を入れる。読者が「全体像」を把握でき離脱率が下がる。

```
## 目次
1. ○○とは？
2. 実際の手順
3. よくある失敗と対策
4. まとめ
```

### 3. 見出し構造（noteのレイアウト最適化）
- `#` タイトル（1つだけ）
- `##` 大見出し（各チャプター）
- `###` 小見出し（手順のステップ、ポイントなど）
- 見出しは「体言止め」または「〜する方法」「〜のポイント」

### 4. 太字・強調の使い方
- **重要キーワード**は太字にする（多用しない。段落に1〜2箇所）
- `> 引用ブロック` で重要な一言・ポイントをハイライト
- 数字は目立たせる（「**3つのステップ**」「**5分で完了**」）

### 5. 箇条書きのルール
- 1点につき1行、30文字以内
- 先頭に絵文字（✅ ⭐ 💡 📌 🔧 ⚠️ など）
- 3〜5点（6点以上は読まれない）
- 箇条書きの前に「**ポイント**」など見出しを入れる

### 6. チュートリアル記事の手順記載
手順がある場合は必ずナンバリング：
```
### Step 1: アカウント作成
### Step 2: APIキーの取得
### Step 3: コードに組み込む
```
各Stepに「何をするか」「なぜ必要か」「具体的な操作」を含める。

### 7. コード・設定値の記載
コードや設定値は必ずコードブロックで囲む：
````
```javascript
const apiKey = 'your_api_key_here';
```
````

### 8. セクション末尾の「まとめボックス」
各大セクションの末尾に要点をまとめる：
```
> 💡 **このセクションのポイント**
> - ○○
> - ○○
> - ○○
```

### 9. NG表現
- 専門用語の乱用（説明なしに使わない）
- 「〜べき」「〜しなければならない」などの命令形
- 「絶対に」「必ず」などの断言
- 文体のブレ（敬体統一：です・ます調）
- 根拠のない「簡単です」「すぐできます」

### 10. まとめセクションの構成
1. この記事で学んだことを3点箇条書き
2. 次のアクション（「まずは○○を試してみてください」）
3. CTA（「他にも○○な動画を投稿しています。チャンネル登録お願いします！」）
